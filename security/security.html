HTTP headers

If you go to This link and enter this site's URL. You can see that my website has a rating of "A+"(maximum security!!). In this page, I will explain the HTTP headers that I have implemented in this website. Make you have create a text file called "_headers" or "netlify.toml" if you are going to use netlify to deploy your sites.

  • Set X-Frame-Options to "DENY", so the site can defend against attacks like clickjacking
  • Set X-XSS-Protection to 1; mode=block,so the site can enables XSS filtering to prevent rendering of the page if an attack is detected.
  • Set X-Content-Type-Options to "nosniff", so it will stop a browser from trying to MIME-sniff the content type and forces it to stick with the declared content-type.
  • Set Referrer-Policy to "strict-origin-when-cross-origin", so The browser will send the full URL to requests to the same origin but only send the origin when requests are cross-origin.
  • Set Content-Security-Policy to "default-src https:", this header is an effective measure to protect the site from XSS attacks and would allow any assets to be loaded over https from any origin.
  • Set Permissions-Policy to "fullscreen=(), geolocation=()", this header allows a site to control which features and APIs can be used in the browser.By specifying an empty origin list, the specified features will be disabled for all documents, including nested documents, regardless of their origin.

Security Principles

After implementing the HTTP headers above, lets talk about what security principles will protect our websites from getting exploited by hackers!

Prevent injection attacks

Make sure the textboxes in the site would allow special characters like";" or ")", so the attackers won't be able to perform injection attacks on the site.

Don't allow weak passwords

Make sure new accounts in this website won't allow weak passwords like 1234. You should make sure a new account’s password needs to meet certain requirements such as “must be at least 12 characters”, “must include special characters”, and “one letter must be capitalized” to eliminate weak passwords for hackers to guess

Eliminate Sensitive Data Exposure

Make sure sensitive data such as last four digits of user credit cards are not being displayed on the browser.We can use encryption libraries such as OpenSSL and libsodium to encrpyt new accounts' passwords

Update softwares

If you ever use any softwares or software libraries on your website, make sure to update them frequently or turn on their auto-update feature. The reason of this is because software developers always patch their softwares' glitches and flaws that may have been exploited by hackers.

Sufficent logging & Monitoring

We need to use monitoring tool to keep track of any earlest warning of problem, so we can improve them before hackers have chance to attack the server. We also need to have server logs to record and identify any unusual activities in the server.
